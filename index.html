CVE-2020-1938

CVE-2020-1938任意文件读取漏洞复现及浅析

Apache Tomcat

Apache 和 Tomcat 都是Apache开源的提供HTTP服务的项目。Tomcat作为轻量级的web服务器得到了广泛的使用，根据ZoomEye统计，全世界大约有4,607,991部署了Tomcat；其中使用的国家统计如下：

而该漏洞可以读取Tomcat目录中webapp下的任意文件，如下图所示，为我在靶机上搭建Tomcat的目录结构

影响范围

Apache Tomcat 6

Apache Tomcat 7<7.0.100

Apache Tomcat 8<8.5.51

Apache Tomcat 9<9.0.31

常见的信息收集工具有很多，比如Kail Linux集成的很多工具，Google hacking等等……此外还有功能异常强大的Shodan和ZoomEye

复现过程

首先我建立了两台虚拟机，一台部署Apache服务器作为靶机，一台作为攻击者，确保在同一个网段下：

靶机部署过程如下：

下载Tomcat

http://archive.apache.org/dist/tomcat

由于Tomcat依赖Java环境，如果没有Java环境，还需要安装Java

https://www.oracle.com/java/technologies/javase-downloads.html

选择Oracle JDK安装就可以，由于同样需要JRE，所以三个都勾选了，然后一路下一步

比较麻烦的主要是环境变量的配置：

我的电脑，右键，属性，高级，环境变量，首先在用户变量下新建变量JAVA_HOME

变量值为C:\\你的目录\Java\jdk-版本号

然后在系统变量Path下添加：%JAVA_HOME%\bin;bin是tomcat存放.sh/.bat文件的目录，是部署Tomcat的目录，然后把jre%JAVA_HOME%\jre\bin同样添加进去

最后打开cmd，输入java -version如果没有报错，那就说明java环境配置无误

之后打开Tomcat/bin目录，并且在此处打开cmd，输入startup

会自动弹出一个java窗口。再在浏览器输入localhost:8080如果出现如下页面说明tomcat成功部署

原理分析

我主要参考了安全客发布的一篇文章：

https://www.anquanke.com/post/id/199347

在tomcat中，conf/server.xml中配置了2个Connector，一个为8080的对外提供的HTTP协议端口，另外一个就是默认的8009 AJP协议端口。此外conf目录还存放了用户名口令等信息。处理ajp请求时候

org.apache.coyote.ajp.AjpProcessor会将ajp内容变成request对象，

我不太懂java，但是据大佬所说Orz：可以控制request对象的三个属性：

javax.servlet.include.request_uri

javax.servlet.include.path_info

javax.servlet.include.servlet_path

然后继续servlet映射流程：

如图所示，servlet就是在Http服务器和数据库之间的一个中间层，查阅相关资料可知，servlet的作用主要有如下五点，详情参见：https://www.runoob.com/servlet/servlet-intro.html

  • 读取客户端（浏览器）发送的显式的数据。这包括网页上的 HTML 表单，或者也可以是来自 applet 或自定义的 HTTP 客户端程序的表单。
读取客户端（浏览器）发送的隐式的 HTTP 请求数据。这包括 cookies、媒体类型和浏览器能理解的压缩格式等等。
处理数据并生成结果。这个过程可能需要访问数据库，执行 RMI 或 CORBA 调用，调用 Web 服务，或者直接计算得出对应的响应。
发送显式的数据（即文档）到客户端（浏览器）。该文档的格式可以是多种多样的，包括文本文件（HTML 或 XML）、二进制文件（GIF 图像）、Excel 等。
发送隐式的 HTTP 响应到客户端（浏览器）。这包括告诉浏览器或其他客户端被返回的文档类型（例如 HTML），设置 cookies 和缓存参数，以及其他类似的任务。

利用方式：

  1. DefaultServlet实现任意文件下载
  2. jspservlet任意后缀文件包含

防范方式：

用不存在该漏洞版本的tomcat

禁用Ajp协议的8009端口

如何美化自己的博客

0x01 美化博客的时机

当然是自己已经搭建完一个初步的博客之后，再进行一些个性化的设置。在这之前一定要做好备份，防止发生玄学错误结果原先的博客也用不了了。。。

搭建博客的教程参见：

https://yueyuehuman.github.io/John_blog/2020/02/14/Github-pages-hexo-to-start-up-your-blog/

正式开始，首先在仓库中新建一个分支，我建的是hexo

0x02 冲鸭！（具体步骤）

参考内容

我参考的是知乎专栏一篇大神写的教程，很详细！

https://zhuanlan.zhihu.com/p/35668237

安装Node.js

下载地址：https://nodejs.org/dist/v9.11.1/node-v9.11.1-x64.msi

作用就是提供一个基本的编译环境，如果成功安装之后，输入node -v 和npm -v 来查看版本号，顺便检查安装成功么得。

安装Git

下载地址：https://git-scm.com/download/win

这就不用说了吧，git是最好用的分布式管理平台（没有之一！）Linus大神做得，管理github的项目非常之方便！

所有的操作都可以一路next，直到最后一个选项选择Use Git from the Windows Command Prompt so that u can run git bash in cmd，然后git –version 查看一下是否安装成功

注册Github帐号

https://github.com/

世界上最大的同性♂交友网站，满足你的一切幻想……先注册帐号

然后在右上角“+”处新建一个新的仓库，创建好仓库之后点进去，选择settings，下拉到Github Pages

安装设置Hexo

首先在本地建立一个新的文件夹用以存放你的博客文件，比如我存的是 C:\ blog directory,我在虚拟机上只建立了一个盘，大家当然可以放在D、E、F……上

在建立好文件夹之后，在文件夹位置右键打开Git Bash，在git bash 中输入：

npm i hexo-cli -g
hexo -v

来进行安装和检查是否安装好Hexo

然后输入：

hexo init
npm install

安装需要的组件，安装完成之后，输入：

hexo g

即generate，用来生成文本

hexo s

即server，然后可以通过访问localhost：4000进行本地预览

本地远程连接Github

输入如下命令，yueyuehuman是我的信息，改成自己的

git config –global user.name “yueyuehuman”
git config –global user.email “yueyuehuman@gmail.com“
ssh-keygen -t rsa -C “yueyuehuman@gmail.com“

然后访问Github Settings，选择SSH and GPG keys，把生成的密钥复制进去

小技巧：

很多人说在输入上述命令时候按照提示信息输入会报错：大概是：Can not find files or directory

所以你在生成密钥时候不要管他提示，直接生成，然后在本地直接寻找./ssh/id_rsa.pub文件就行，然后用记事本打开文件，把所有的信息直接复制进去就行，之后输入：

ssh -T git@github.com

来测试一下是否成功，如果成功如下图：

如果解决了上述错误，又出现什么cannot entry block，说明_config.yml文件中格式有误，这个是Hexo的一大bug！它解析语句时候必须遵循

类型名+冒号+空格+参数，有些语句可能没空格，就会导致错误，因此需要进.yml文件中自己排查。

然后打开博客根目录下_config.yml文件，修改最后一行配置，同样的改成自己的仓库名：

deploy：
type：git
repository： https://github.com/yueyuehuman/John_blog
branch：master

0x03 发布你的博客

首先在博客根目录下右键打开git bash，安装一个扩展

npm i hexo-deployer-git

然后输入hexo new post “你的文章名”，新建一篇文章。

然后打开C：\blog\source_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章。

编写完markdown文件后，根目录下输入hexo g生成静态网页，输入hexo d上传到github上。这时打开你的github.io主页就能看到发布的文章啦。

小提示：

当然很多人发现可以上去了，但是不加载js和css之类的文件，太丑了，还是打开_config.yml

在url和root中分别加入你的博客地址和仓库名，博客地址就是在githubpages中提示的，仓库名就是最上面的，格式一定严格按照下图，缺少引号或者前后的斜线都无法被识别，然后重新上传，效果如下：

0x01 为什么要建立自己的博客

作为一个程序猿，每天的大部分精力都是用来解决问题，很多问题可能花费了很长时间才得以解决，那么通过合理的方式记录下来最好了！有什么好的方式呢？CSDN、简书、知乎都提供了博客，但是一些博客有可能难以通过审核，自由度太低。而blogger之类的呢，你的解决方案无法被那些没法科学上网的人看见。通过Typora以Markdown形式在本地记录，又不方便。因此为什么不自己搭一个支持Markdown编写模式的博客呢？GithubPage+Hexo就是一个很好的解决方案。Github不多说了，世界上最大的同性交友网站，访客量很大。在该平台上维护的博客也相对稳定。由于自己不是前端大神，因此完全自己整界面也难，发现Hexo很符合我的taste，简约！
