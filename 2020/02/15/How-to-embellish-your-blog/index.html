如何美化自己的博客

0x01 美化博客的时机

当然是自己已经搭建完一个初步的博客之后，再进行一些个性化的设置。在这之前一定要做好备份，防止发生玄学错误结果原先的博客也用不了了。。。

搭建博客的教程参见：

https://yueyuehuman.github.io/John_blog/2020/02/14/Github-pages-hexo-to-start-up-your-blog/

正式开始，首先在仓库中新建一个分支，我建的是hexo

0x02 冲鸭！（具体步骤）

参考内容

我参考的是知乎专栏一篇大神写的教程，很详细！

https://zhuanlan.zhihu.com/p/35668237

安装Node.js

下载地址：https://nodejs.org/dist/v9.11.1/node-v9.11.1-x64.msi

作用就是提供一个基本的编译环境，如果成功安装之后，输入node -v 和npm -v 来查看版本号，顺便检查安装成功么得。

安装Git

下载地址：https://git-scm.com/download/win

这就不用说了吧，git是最好用的分布式管理平台（没有之一！）Linus大神做得，管理github的项目非常之方便！

所有的操作都可以一路next，直到最后一个选项选择Use Git from the Windows Command Prompt so that u can run git bash in cmd，然后git –version 查看一下是否安装成功

注册Github帐号

https://github.com/

世界上最大的同性♂交友网站，满足你的一切幻想……先注册帐号

然后在右上角“+”处新建一个新的仓库，创建好仓库之后点进去，选择settings，下拉到Github Pages

安装设置Hexo

首先在本地建立一个新的文件夹用以存放你的博客文件，比如我存的是 C:\ blog directory,我在虚拟机上只建立了一个盘，大家当然可以放在D、E、F……上

在建立好文件夹之后，在文件夹位置右键打开Git Bash，在git bash 中输入：

npm i hexo-cli -g
hexo -v

来进行安装和检查是否安装好Hexo

然后输入：

hexo init
npm install

安装需要的组件，安装完成之后，输入：

hexo g

即generate，用来生成文本

hexo s

即server，然后可以通过访问localhost：4000进行本地预览

本地远程连接Github

输入如下命令，yueyuehuman是我的信息，改成自己的

git config –global user.name “yueyuehuman”
git config –global user.email “yueyuehuman@gmail.com“
ssh-keygen -t rsa -C “yueyuehuman@gmail.com“

然后访问Github Settings，选择SSH and GPG keys，把生成的密钥复制进去

小技巧：

很多人说在输入上述命令时候按照提示信息输入会报错：大概是：Can not find files or directory

所以你在生成密钥时候不要管他提示，直接生成，然后在本地直接寻找./ssh/id_rsa.pub文件就行，然后用记事本打开文件，把所有的信息直接复制进去就行，之后输入：

ssh -T git@github.com

来测试一下是否成功，如果成功如下图：

如果解决了上述错误，又出现什么cannot entry block，说明_config.yml文件中格式有误，这个是Hexo的一大bug！它解析语句时候必须遵循

类型名+冒号+空格+参数，有些语句可能没空格，就会导致错误，因此需要进.yml文件中自己排查。

然后打开博客根目录下_config.yml文件，修改最后一行配置，同样的改成自己的仓库名：

deploy：
type：git
repository： https://github.com/yueyuehuman/John_blog
branch：master

0x03 发布你的博客

首先在博客根目录下右键打开git bash，安装一个扩展

npm i hexo-deployer-git

然后输入hexo new post “你的文章名”，新建一篇文章。

然后打开C：\blog\source_posts的目录，可以发现下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章。

编写完markdown文件后，根目录下输入hexo g生成静态网页，输入hexo d上传到github上。这时打开你的github.io主页就能看到发布的文章啦。

小提示：

当然很多人发现可以上去了，但是不加载js和css之类的文件，太丑了，还是打开_config.yml

在url和root中分别加入你的博客地址和仓库名，博客地址就是在githubpages中提示的，仓库名就是最上面的，格式一定严格按照下图，缺少引号或者前后的斜线都无法被识别，然后重新上传，效果如下：
